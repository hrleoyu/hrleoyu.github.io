浅析MVVM的双向绑定


MVVM的概念

MVVM是Model-View-ViewModel的简写，是一个软件架构设计模式。而在MVVM框架中，View(视图)和Model(数据)是不可以直接通讯的，在他们之间存在着ViewModel这个中间介充当着观察者的角色。当用户操作View，ViewModel感知到变化然后传递给Model发生相应的改变，反之亦然。这一来一回就是我们所说的双向绑定。



  • view层

view是视图层，也就是用户所看到的界面。前端主要通过程序员写Html和Css来构建，为了能够更方面地展现viewmodel 或者 model 层的数据，现在已经产生了各种各样的前后端模板语言，比如freemarker、Marko、pug等。

  • model层

model是数据模型，主要是用于后端进行的各种业务逻辑处理和数据操控。

  • viewmodel层

viewmodel是由前端开发程序员组织生成和维护的视图数据层。MVVM模式的核心，是通过这一层连接view、model层。

MVVM的适用范围

最大的优势就是编写前端逻辑非常复杂的页面，尤其是需要大量dom操作的逻辑，利用MVVM可以极大的简化前端页面的逻辑。
但是MVVM不是万能的，它的目的是为了简化复杂的前端逻辑。但是如果要通过被搜索引擎索引的页面，就无法获取通过 api 加载的数据了，比如新闻、博客、文档等。

实现MVVM的双向绑定

了解Object.defineProperty()

这个 api 是实现双向绑定的核心，最主要的作用是重写数据的 get 、 set 方法。
语法
Object.defineProperty(obj,prop,descriptor)
obj：要处理的目标对象
prop：要定义或修改的属性名称
descriptor：将被定义或修改的属性描述符

数据描述符

  • value: 是这个属性的值。 默认是 undefined。
  • writable : 当这个属性为 true 时，这个属性才能被修改，默认是 false。

存取描述符

  • get 一个给属性提供getter的方法，如果没有getter则为undefined。
  • set 一个给属性提供setter的方法，如果没有setter则为undefined。


画一下重点

  • get,set设置时不能设置writable和value，交叉设置或者同时存在都会报错
  • 通过 defineProperty 设置的属性，默认不能删除，不能遍历，但是可以通过设置更改他们
  • get,set是函数，可以做很多事情
